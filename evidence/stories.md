## Story S1: Text contrast
**Situation**: When I'm using the program in an environment with bright light\
**Motivation**: I want the text to still be clearly visible\
**Outcome**: So I can still be sure in what I am doing\
**Underlying need**: Because otherwise I may misread or mistype tasks

**Inclusion risk**: Cognitive\
**Type**: Job story\
**WCAG**: 1.4.3 Contrast (AA)

---

## Story S2: Confirmation Feedback
**Situation**: When I submit a form (add/edit/delete task)\
**Motivation**: I want immediate, explicit confirmation that the action succeeded\
**Outcome**: So I can trust the interface without refreshing to verify\
**Underlying need**: Because uncertainty about save status causes anxiety and inefficient workarounds

**Inclusion risk**: Cognitive, Screen reader, Low digital literacy\
**Type**: Job story\
**WCAG**: 4.1.3 Status Messages (AA)

---

## Story S3: Keyboard navigation
**Situation** When I navigate the website using only a keyboard\
**Motivation**: I want to be able to access all the features of the website as effectively as when using a mouse\
**Outcome**: So I have more methods to interact with the website\
**Underlying need**: Because I should be able to use the website with whatever input method works best for me

**Inclusion risk**: Feedback, Efficiency\
**Type**: Job story\
**WCAG**: 2.1.1 Keyboard (A)

---

## Story S4: Screen reader
**Situation**: When using a screen reader to interpret the contents of the website\
**Motivation**: I want to be able to get a good general overview of the websites functions and how to access them\
**Outcome**: So I can quickly navigate to the features I want to use without floundering for long\
**Underlying need**: Because if the layout of the website is not communicated effectively it will be difficult for me to use

**Inclusion risk**: Keyboard, Cognitive\
**Type**: Job story\
**WCAG**: N/A

---

## Story S5: Feature clarity
**Situation**: When using the task manager with a keyboard and mouse\
**Motivation**: I want to be able to perform the basic features of the task manager (adding/deleting tasks, view tasks) within one or two actions form visiting the webpage\
**Outcome**: So use of the website is as efficient and easy as possible\
**Underlying need**: Because having all the features easily accessible from the landing page communicates the functions and purpose of the website better

**Inclusion risk**: Screen reader, Keyboard, Accessibility\
**Type**: Job story\
**WCAG**: N/A
